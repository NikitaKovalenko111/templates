=== ЦЕЛЬ ===

Связать frontend и backend:

- через существующие API
- без изменения архитектуры
- без дублирования логики

---

=== ПРАВИЛА ===

1. Backend — источник истины
    - не менять API без необходимости
    - использовать существующие endpoints

2. API слой на фронте:
    - создать отдельный слой (api/)
    - все запросы идут только через него
    - не делать fetch/axios прямо в компонентах

3. Работа с данными:
    - соответствие backend моделям
    - не придумывать новые поля
    - не трансформировать данные без причины

4. React:
    - минимально необходимый state
    - использовать useEffect для загрузки данных

5. Обработка ошибок:
    - базовая обработка (try/catch)
    - не игнорировать ошибки API

6. Чистота:
    - не дублировать запросы
    - не смешивать UI и API логику

---

=== ЗАПРЕЩЕНО ===

- менять backend API без причины
- делать запросы напрямую в компонентах
- усложнять архитектуру
- дублировать бизнес-логику с backend

---

1. Какие endpoints используются
